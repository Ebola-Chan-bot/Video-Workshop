面向本地视频处理的 PowerShell 模块，提供三个常用视频处理命令：

- 烧录字幕：将第一个字幕轨硬烧进第一个视频轨，并复制其它未参与烧录的轨道。
- DV转HDR10：用 libplacebo 处理 Dolby Vision 色彩信息，输出带 HDR10 元数据的 HEVC 文件。
- 超级合并：按列表或剪贴板顺序合并视频片段，兼容时 copy，不兼容时只标准化必要轨道。

# 安装

从 PowerShell Gallery 安装：

```powershell
Install-Module -Name 视频工坊
Import-Module 视频工坊
Get-Command -Module 视频工坊
```

# 快速使用

```powershell
Import-Module .\视频工坊.psd1
Get-Command -Module 视频工坊
```

```powershell
烧录字幕 -输入文件 'D:\电影.mkv' -字幕序号 0
DV转HDR10 -输入文件 'D:\电影.DV.mkv' -测试秒数 10
超级合并 -输出文件 '.\merged.mkv' -列表文件 '.\要合并的文件列表.txt'
```

# 命令说明

## 烧录字幕

```powershell
烧录字幕 -输入文件 'D:\电影.mkv' [-输出文件 'D:\电影.hardsub.mkv'] [-字幕序号 0]
```

- 选取输入文件的第一个视频轨和选定的（默认首个）字幕轨。文本字幕使用 ffmpeg `subtitles` 滤镜渲染；DVD/PGS/DVB/XSub 等位图字幕会缩放到视频尺寸后 `overlay` 到画面上。
- 输出视频会重新编码为 HEVC：优先 `hevc_nvenc`（CQ 23），不可用时回退 `libx265`（CRF 23、medium）。音频轨和未被烧录的其它轨道会 copy。
- 未指定 `-输出文件` 时，默认在原目录生成 `原文件名_烧录字幕 + 原扩展名`。输出已存在时会覆盖。
- 输入必须包含至少一个视频轨和一个字幕轨；缺少轨道会报错。硬字幕是不可逆处理，生成后字幕成为画面的一部分。

## DV转HDR10

```powershell
DV转HDR10 -输入文件 'D:\电影.DV.mkv' [-输出文件 'D:\电影.HDR10.mkv'] [-质量 19] [-测试秒数 30]
```

- 需要支持 `libplacebo` 滤镜的 ffmpeg。命令会把 Dolby Vision 色彩处理为 BT.2020、PQ（SMPTE ST 2084）、TV range 的 HDR10 输出，并写入对应色彩元数据。
- 只转换第一个视频轨，所有音频轨会 copy；字幕、附件和其它非音频轨不会输出。
- 优先使用 `hevc_nvenc`（preset p4、VBR、CQ=`-质量`），没有 NVENC 时使用 `libx265`（medium、CRF=`-质量`）。`-质量` 范围为 1 到 51，数值越小质量越高、文件通常越大，默认 19。
- 未指定输出路径时，默认生成 `原文件名_HDR10.mkv`。设置 `-测试秒数` 后只转换开头指定秒数，默认输出名变为 `原文件名_HDR10_testNs.mkv`，适合先检查色彩和编码速度。

## 超级合并

```powershell
超级合并 -输出文件 'D:\合集.mkv' -列表文件 'D:\要合并的文件列表.txt' [-不兼容策略 Min|Max] [-详细模式]
```

- 输入列表支持普通路径、带引号路径和 ffmpeg concat 形式的 `file '路径'`；空行和 `#` 注释会被忽略。显式传入 `-列表文件` 时按 UTF-8 读取；没有显式传入时会从剪贴板读取路径列表。
- 命令先用 ffprobe 探测每段的第一个视频轨、音频轨和字幕轨，并验证所有输入文件存在且至少有视频轨。
- 如果所有输入的流数量相同，并且每个流索引位置的 `codec_type`/`codec_name` 相同，会进入同构快速路径：用 mkvmerge 按真实视频包 PTS 精剪每段，再修复 H.264 SPS/PPS 容器元数据，最后用 ffmpeg concat demuxer 无损拼接。这个判定只看轨道布局和编码名，不会额外比较分辨率、帧率或像素格式。
- 如果不能走同构快速路径，会进入标准化路径。视频比较编码、宽度、高度、像素格式和帧率；兼容就 copy，不兼容时只处理不一致的片段。只有分辨率不同且当前视频编码支持同一轨道中途变更分辨率时，会保留各段原分辨率以避免重编码；当前按 H.264/HEVC/AV1/VP9 判断。若还因为编码、像素格式或帧率等其它原因必须重编码，则宽高/帧率按 `-不兼容策略` 选择 Min 或 Max，缩放时保持比例并补边，编码器优先 GPU（NVENC/QSV/AMF），否则回退 CPU。
- 音频比较编码、采样率和声道数；兼容就 copy，不兼容或缺失时会补静音并重编码。通常输出 AAC；`-不兼容策略 Max` 且存在无损音频时，会优先使用 FLAC/ALAC 这类无损目标，避免无损源被转为有损。
- 字幕支持 ASS/SRT 统一和空字幕占位。位图字幕不能与文本字幕互转，也不能为空段生成等价位图占位；因此存在位图字幕时，所有片段必须都有同一种位图字幕，否则命令会报错。
- 输出容器会综合用户扩展名、输入容器、字幕和音频兼容性选择；必要时会切换到 MKV。若用户显式指定了其它扩展名，最终文件名可能保持用户扩展名，但命令会提示实际容器。
- 默认会清理临时目录。使用 `-详细模式` 会显示外部命令输出并保留临时目录，方便检查分段文件和 concat 列表。

# 依赖

- ffmpeg
- ffprobe
- mkvmerge（超级合并需要）

模块会复用同一套依赖检测、外部命令调用、媒体探测、字幕占位和容器修正过程。缺少 ffmpeg/ffprobe/mkvmerge 时，相关命令会尝试通过 winget 或提权安装脚本自动安装，并刷新当前会话的 PATH。

PowerShell 内置帮助也包含更细的参数和示例：

```powershell
Get-Help 烧录字幕 -Detailed
Get-Help DV转HDR10 -Detailed
Get-Help 超级合并 -Detailed
```
